# Screenshot Shot List

> 注: 本清单基于旧版带登录入口的界面，当前产品已经切换为单用户工作区模式。若后续重新拍图，请以聊天页、设置页和 schema 视图为准。

这个文件用于统一 README 截图替换标准。

## 输出文件名

保持这些文件名不变，这样替换图片后不用再改 README:

- `docs/images/login.png`
- `docs/images/chat.png`
- `docs/images/semantic.png`
- `docs/images/schema.png`

## 推荐截图内容

### `login.png`

- 展示登录/注册入口
- 页面干净，不带浏览器书签栏
- 不要出现真实邮箱、真实 token、真实域名

### `chat.png`

- 展示聊天主界面
- 左侧有历史列表
- 中间至少显示:
  - 用户问题
  - SQL 结果
  - 图表或 Python 输出
  - 诊断面板
- 诊断区建议能看到:
  - 数据库连接
  - provider
  - 自动修复次数
  - 尝试记录

### `semantic.png`

- 展示语义层或业务术语配置页
- 最好能看出术语、别名、说明

### `schema.png`

- 展示 Schema 布局或表关系视图
- 画面里至少出现 3 个以上表节点和关系线

## 统一规范

- 推荐浅色主题
- 浏览器宽度建议 `1440px`
- 截图比例尽量统一
- 示例数据用演示数据，不要出现真实客户名、手机号、地址、API Key
- 尽量保留产品真实状态，不要做假数据拼图

## 我建议的最终 README 截图顺序

1. 登录页
2. 聊天工作台
3. 语义层
4. Schema 关系图

## 备注

如果后面要换成更强的 GitHub 首屏效果，优先补一张新的 `chat.png`:

- 顶部显示模型和连接状态
- 中间显示分析总结
- 下方显示图表
- 右侧或下方显示诊断轨迹

这一张图最值钱。
